# 咨询证据型内容档案 v1

仅当 `deck_plan.json.style.content_profile` 为
`consulting-evidence-cn-v1` 时使用本文件。它规定内容如何从来源走向结论，不是视觉模板：
不得替换 `visual_style`、GHB 品牌色、字体、封面、母版或构建流程。

## 目标

让每个关键页面形成清晰链路：**结论 → 可追溯证据 → 业务含义**。

- 结论回答“所以什么”，而不是重复章节名；
- 证据给出比较基准、数字口径和来源；
- 含义明确标为推断或建议，不能伪装成来源事实；
- 内容语义决定 SVG 构图，不能为了“咨询感”强套矩阵、漏斗或蓝色图表。

## 叙事顺序

根据来源选择并压缩为以下骨架，不要求固定页数：

```text
总判断
  → 现状与关键差距
  → 分群、场景或时间比较
  → 价值是否兑现
  → 驱动机制或标杆实践
  → 对当前受众的行动、约束与风险
```

若来源不支持标杆比较，应跳过该段；不得虚构“高绩效者”或因果结论。

## 标题和逐页内容

先写 `message`，再选择图形。优先使用：

| 情境 | 标题形式 |
|---|---|
| 现状存在落差 | `X 已经…，但 Y 仍…` |
| 分群存在差异 | `A 比 B 更…，差异体现为…` |
| 驱动因素 | `要实现 Z，关键先做 A/B/C` |
| 行动建议 | `下一步应…，以解决…` |

不要使用“市场现状”“研究发现”“高绩效企业”这类无结论标题。源资料只有主题而没有
事实时，将其放在 `items` 或附录，不能补造结论。

事实型页面可在现有 slide 对象中增加 `consulting_content`：

```json
{
  "message": "AI 应用已普及，但规模化仍是企业级价值的主要缺口",
  "source_refs": ["sources/source.md#图1-AI应用阶段"],
  "consulting_content": {
    "claim_type": "fact",
    "evidence": [{
      "label": "至少一个职能常态化使用 AI",
      "value": "88%",
      "source_ref": "sources/source.md#图1-AI应用阶段"
    }],
    "so_what": {
      "type": "inference",
      "text": "下一阶段应将讨论从单点试验转向规模化条件。"
    }
  }
}
```

### 字段规则

- `claim_type` 只能是 `fact`、`inference` 或 `recommendation`。
- `fact` 至少包含一条 `evidence`；每条证据都需要非空的 `label`、`value`、
  `source_ref`，且 `source_ref` 必须出现在该页 `source_refs` 中。
- `so_what` 可选；其 `type` 只能是 `inference` 或 `recommendation`，并需要
  非空的 `text`。
- 锚点页、过渡页或没有量化事实的流程页可省略 `consulting_content`，但仍要保留
  现有来源要求。

## 证据与版式选择

- 一页只服务一个主结论；默认一个主图、主表或主结构，突出 1–3 个决定性数字。
- 优先展示能改变判断的比较：时间趋势、分群差异、目标差距、阶段漏斗或前后变化。
- `fact` 保留单位、时点、分母、样本和限定词；`inference` 使用“表明 / 意味着 /
  提示”；`recommendation` 使用“建议 / 应考虑”。
- 密集研究表格拆页或放附录，不通过缩小字体搬运。
- 页面底部给出精简来源行；外部研究至少保留报告名、年份和图号或页码。

| 关系 | 推荐现有结构 | 不应使用 |
|---|---|---|
| 趋势与阶段差距 | `data-story` / `editorial` 或 `waterfall` | 为了风格硬塞矩阵 |
| 可比群组 | `comparison` / `matrix` 或并列条形比较 | 没有共同维度的卡片堆叠 |
| 行动链路 | `process` / `swimlane` / `layered_arch` | 用并列 bullet 伪装流程 |
| 章节结论 | `summary` / `hero` / `section-anchor` / `editorial` | 整页图片代替可编辑内容 |

## 交付前检查

1. 每页标题是否表达结论；
2. 每个事实是否有数字、口径和 `source_refs`；
3. 每个推断/建议是否与事实明确区分；
4. 图形是否解释比较关系，而非只装饰；
5. 现有 SVG、可编辑性、溢出、渲染和人工逐页检查是否仍全部通过。
