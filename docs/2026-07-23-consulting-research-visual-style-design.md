# 咨询研究型视觉生成风格设计（v2）

## 问题与结论

v1 的 `consulting-evidence-cn-v1` 只控制“结论—证据—含义”，并且明确禁止改变 GHB
视觉。因此，即使内容采用咨询叙事，成品仍会呈现银行红色、圆角卡片和模板页眉。这个
边界保护了默认行为，却无法满足用户明确要求的“麦肯锡式视觉”。

本次增加独立、显式选择的 `consulting-research-cn-v1`。它参考附件可见的研究报告规律，
将其转成 16:9 正文视觉合同；不复制品牌资产。默认不启用，所以现有 GHB 项目和仅使用
内容档案的项目不会改变。

## 附件依据与规则映射

| 附件可见证据 | 生成规则 |
|---|---|
| PDF 第 4、5 页以白底、黑色结论标题、留白和细分隔线承载正文 | 正文采用全幅白底和细灰规则线；不使用模板米色底、红色页眉或圆角外框 |
| PDF 第 5、13 页以蓝/灰横向比较、直接数字标签表达群组差异 | 图表默认灰作基线、海军蓝作比较、亮蓝作强调；数值直接贴近条形或节点 |
| PDF 第 13、15 页让证据占据主要面积，正文只解释必要含义 | 一页一个主证据结构，文字只承担口径、解释和推断，不再堆砌模块卡片 |
| 每页底部保留注释、资料来源和页码 | 每张正文页固定来源/口径脚注与页码，并以细线与主体隔开 |
| 报告标题呈衬线、图表标签呈无衬线 | 标题采用 `Songti SC/SimSun`，正文和图表采用 `Source Han Sans SC` |

## 接口与兼容性

```json
{
  "style": {
    "content_profile": "consulting-evidence-cn-v1",
    "visual_profile": "consulting-research-cn-v1"
  }
}
```

- `content_profile` 继续管叙事与证据边界；`visual_profile` 只管正文视觉，两者可独立选择。
- `visual_profile` 缺失时，投影器不会生成字段、不会更改 `spec_lock.md`，GHB 默认路径不变。
- 未知 profile 在写任何投影文件前失败；选中的 profile 被写入确认摘要、布局计划、设计简报和执行锁，避免后续 SVG 创作遗失。
- 正文覆盖 GHB 母版仅限选中的 profile；封面与致谢页仍保留 GHB，以维持现有模板、OOXML 合并和交付链的兼容性。

## 验收标准

1. 默认 Standard fixture 仍不含 `visual_profile`，且所有既有测试通过。
2. 选择该 profile 后，生成的 `layout_plan.json`、`design_spec.md` 和 `spec_lock.md` 可检索到同一 profile；锁中包含研究型配色、字体和全幅正文覆盖规则。
3. 成品正文不出现 GHB Logo/页眉、银行红色卡片或米色纹样；可见白底、衬线标题、细线、蓝灰证据关系、来源与页码。
4. 每页仍通过 SVG、PPTX、字体、渲染与回读检查，且正文保持 DrawingML 可编辑。
